### 4.2 核心功能详细说明

#### 4.2.1 用户登录

**功能描述**：用户通过用户名和密码登录系统

**API接口**：`POST /api/user/login/`

**请求示例**：
```json
{
  "username": "admin",
  "password": "admin123"
}
```

**响应示例**：
```json
{
  "status": "success",
  "user": {
    "id": 1,
    "username": "admin",
    "email": "admin@example.com",
    "role": "admin"
  }
}
```

**实现文件**：`backend/user/views.py`

---

#### 4.2.2 URL检测

**功能描述**：检测URL是否为钓鱼网站

**API接口**：`POST /api/detection/detect-url/`

**请求示例**：
```json
{
  "url": "https://phishing.example.com/login"
}
```

**响应示例**：
```json
{
  "status": "success",
  "data": {
    "is_phishing": true,
    "confidence": 0.85,
    "message": "该URL存在钓鱼风险"
  }
}
```

**实现文件**：`backend/detection/views.py`

**检测算法**：
1. 查询钓鱼特征库获取URL特征规则
2. 使用正则表达式匹配URL
3. 根据匹配的特征计算置信度
4. 置信度 >= 0.6 判定为钓鱼

---

#### 4.2.3 邮件分析

**功能描述**：分析邮件头识别钓鱼邮件

**API接口**：`POST /api/detection/analyze-email/`

**请求示例**：
```json
{
  "email_header": "From: noreply@fake-bank.com\r\nTo: user@university.edu\r\nSubject: 账户安全提醒"
}
```

**响应示例**：
```json
{
  "status": "success",
  "data": {
    "is_phishing": true,
    "confidence": 0.92,
    "risk_factors": ["可疑发件人", "伪装域名"]
  }
}
```

**实现文件**：`backend/detection/views.py`

## 六、常见问题

### 6.1 前端无法访问后端API

**问题现象**：前端调用API时出现CORS错误

**解决方案**：
1. 确保后端服务已启动（http://localhost:8000）
2. 检查 `settings.py` 中的CORS配置：
```python
CORS_ALLOWED_ORIGINS = [
    "http://localhost:5173",
    "http://127.0.0.1:5173",
]
```

### 6.2 数据库连接失败

**问题现象**：Django启动时无法连接数据库

**解决方案**：
1. 确保 `db.sqlite3` 文件存在
2. 检查数据库目录权限
3. 运行数据库迁移：
```bash
python manage.py migrate
```

### 6.3 登录失败

**问题现象**：用户名密码正确但登录失败

**解决方案**：
1. 检查后端服务是否正常运行
2. 检查用户是否已激活（is_active=True）
3. 使用Django管理后台创建测试用户：
```bash
python manage.py createsuperuser
```

```javascript
export default defineConfig({
  plugins: [react()],
  server: {

      '/api': {
        target: 'http://localhost:8000',
        changeOrigin: true
      }
    }
  }
```

```python
# 数据库配置
```
