### 1.2 分层架构表格

| 层级 | 名称 | 技术栈 | 职责 |
|:---:|:---|:---|:---|
| L1 | **表现层** | React 18 + Ant Design 5 + React Router v6 + ECharts 5 | UI交互、页面路由、数据可视化、用户交互处理 |
| L2 | **业务逻辑层** | Django 4.2 + Django REST Framework 3.14 | 权限验证、业务规则处理、核心算法实现、数据校验 |
| L3 | **数据访问层** | Django ORM | 封装数据库增删改查操作，提供数据持久化接口 |
| L4 | **基础设施层** | SQLite 3 + CORS中间件 + 日志系统 | 数据存储、跨域支持、日志记录、静态文件服务 |

## 三、层间接口规范

### 3.1 接口规范表格

| 层级交互 | 通信协议 | 数据格式 | 接口规范 | 示例 |
|:---|:---|:---|:---|:---|
| 表现层 → 业务逻辑层 | HTTP/HTTPS | JSON | RESTful API | POST /api/user/login/ |
| 业务逻辑层 → 数据访问层 | Python对象 | ORM对象 | Django ORM API | User.objects.get() |
| 数据访问层 → 基础设施层 | SQL | 数据库记录 | SQLite驱动 | SELECT * FROM users |

### 3.2 数据传输格式

#### 请求格式

```json
{
  "method": "POST",
  "url": "/api/detection/detect-url/",
  "headers": {
    "Content-Type": "application/json"
  },
  "body": {
    "url": "https://example.com"
  }
}
```

#### 响应格式

```json
{
  "status": "success",
  "data": {
    "is_phishing": false,
    "confidence": 0.25
  },
  "message": "检测完成"
}
```
